# Events and Intents

All communication in Polyverse uses the `Event` enum defined in `libs/kernel/src/event.rs`.

## `Event::Raw(RawEvent)`

The payload generated by external adapters (Discord, Telegram).

```rust
pub struct RawEvent {
    pub platform: Platform,
    pub channel_id: String,
    pub message_id: String,
    pub user_id: String,
    pub username: String,
    pub content: String,
    pub is_mention: bool,
    pub is_dm: bool,
    pub timestamp: DateTime<Utc>,
}
```

## `Event::Intent(IntentEvent)`

The structured classification generated after a `RawEvent`.

```rust
pub struct IntentEvent {
    pub source: RawEvent,
    pub intent: Intent,
    pub sentiment: Sentiment,
    pub needs_cloud: bool,
    pub confidence: f32,
}
```

**Intent Variants:**
- `Command`
- `Question`
- `ChitChat`
- `Insult`
- `Noise`
- `ComplexQuery`

**Sentiment Variants:**
- `Positive`
- `Neutral`
- `Negative`

## `Event::Response(ResponseEvent)`

The payload generated by cognitive workers for text output.

```rust
pub struct ResponseEvent {
    pub platform: Platform,
    pub channel_id: String,
    pub reply_to_message_id: Option<String>,
    pub reply_to_user: Option<String>,
    pub is_dm: bool,
    pub content: String,
    pub source: ResponseSource,
}
```

**ResponseSource Variants:**
- `LocalSLM`
- `CloudLLM`
- `Template`

## `Event::BotTurnCompletion(BotTurnCompletion)`

Emitted to signal the end of a response cycle.

```rust
pub struct BotTurnCompletion {
    pub platform: Platform,
    pub channel_id: String,
    pub reply_to_message_id: Option<String>,
    pub reply_to_user: Option<String>,
    pub content: String,
}
```

## `Event::Biology(BiologyEvent)`

Signals a change to the agent's internal biological state.

```rust
pub struct BiologyEvent {
    pub kind: BiologyEventKind,
    pub timestamp: DateTime<Utc>,
}
```

**BiologyEventKind Variants:**
- `EnergyChanged { delta: f32, reason: String }`
- `MoodChanged { new_mood: String, trigger: String }`
- `SleepStarted`
- `SleepEnded`

## `Event::System(SystemEvent)`

Runtime lifecycle signals broadcast by the `Supervisor` or `Coordinator`.

```rust
pub enum SystemEvent {
    WorkerStarted { name: String },
    WorkerStopped { name: String },
    WorkerError { name: String, error: String },
    ShutdownRequested,
    HealthCheckRequest,
}
```
