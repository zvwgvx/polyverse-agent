# Reference

This section is for durable lookup material rather than narrative explanations.

Use it when you already understand the broad shape of the repository and need exact files, toggles, commands, or test layers.

## Pages in this section

- [Configuration](./configuration.md) — config files, env vars, runtime layering, and default local binds
- [Testing](./testing.md) — test layers, important test paths, and common commands

## What belongs here

- exact command references
- config file locations
- environment variable groups
- stable test entrypoints

## What does not belong here

- speculative future design
- implementation proposals
- long-form architecture narrative

For those topics, use [Overview](../overview/) or [Architecture](../architecture/) instead.
